# 后端岗位线上测试题

## 📘 背景说明

你将获得以下两个 GitHub 仓库地址：

1. 前端项目仓库（Frontend）[https://github.com/xuelinc91-creator/Live](https://github.com/xuelinc91-creator/Live)

2. 前后端网关项目仓库 （Gateway，用于前端与后端通信）[https://github.com/xuelinc91-creator/live-gateway](https://github.com/xuelinc91-creator/live-gateway)

你的任务是：按照你自己的理解为该项目编写一个后端服务（可以使用任意主流后端框架），使用mock 数据模拟真实业务逻辑，并将整个项目部署上线，最终提供一个可在线访问的演示链接。

## 🎯 开发要求

### 1. 后端项目开发

- 语言与框架不限（推荐使用 Node.js/Express、NestJS、Spring Boot、Django、Flask、Go Gin 等）

- 使用 mock 数据（不要求连接真实数据库）

- 实现前端项目所需的主要接口，使前端功能可完整演示

- 接口数据需返回合理的结构（JSON 格式），示例：

```JSON

{
"code": 0,
"message": "success",
"data": { ... }
}
```

### 2. 与前端/网关的集成

- 根据提供的 前端项目 与 网关项目 文档，确保接口路径、跨域配置、鉴权逻辑等匹配正常

- 你可以直接运行前端项目并本地调试你的后端

### 3. 部署要求

- 将前端、网关、后端都运行在同一台服务器上（或云服务平台），淘宝有廉价服务器

- 任选一个公开可访问的平台部署（例如：Vercel、Render、Netlify、Railway、AWS、阿里云、腾讯云、Cloudflare Pages 等）

- 需提供一个在线演示地址（可访问前端页面）

- 如果有自己不能实现的部分请在readme.md作说明

### 4. 加分项目

- 使用vercel、cloudflare等服务成功部署运行项目
